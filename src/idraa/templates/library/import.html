Field guidance

  • The file must be a JSON array of entry objects — identical in shape to the seed library bundle (see the template).
  • slug — required, unique across the whole library. A duplicate slug is skipped, not overwritten.
  • status — must be published; imports go live immediately.
  • threat_event_type, threat_actor_type, asset_class — must be valid enum values.
  • threat_event_frequency, vulnerability, primary_loss, secondary_loss — PERT objects with finite low ≤ mode ≤ high (Secondary Loss is optional).
  • description and canonical_fair_gap — required, 20+ characters each.
