{# import_preview.html — Mobile tranche 2e. Card-stacking table macro for the two-step import preview pages (scenarios / overlays / library bundle). Each preview page shows a parsed-row table that ends in an action badge, plus an optional validation-errors table. This macro renders BOTH a desktop (hidden md:block) AND a mobile card stack (md:hidden), so the dense rows reflow to readable cards on phones instead of forcing horizontal scroll (the old overflow-x-auto behaviour). Args: rows: list[dict] — each a parsed preview row or validation-error row. columns: list[dict], in display order: label: str — header cell + mobile
label key: str — dict key on the row mono: bool optional — font-mono (numbers / codes); default off align: 'right' optional — right-align the desktop cell; default left action_key: str optional — the column key whose value is an import action (create / add / update / skip / error). That cell renders as a DaisyUI badge, and on mobile it is pulled to the card's top-right as the row's visual anchor. Omit for tables with no action column (e.g. the validation-errors table). Row contract: rows are plain dicts. Jinja subscript returns Undefined for a missing key, so `row[c.key]|default('')` is safe for optional columns. The Jinja env uses the default (non-strict) Undefined, so a column dict may omit `mono`/`align` and they read as falsy. #} {% macro _action_badge(action) -%} {%- set _styles = { 'create': 'badge-success', 'add': 'badge-success', 'update': 'badge-info', 'skip': 'badge-ghost', 'error': 'badge-error', } -%} {{ action }} {%- endmacro %} {% macro preview_table(rows, columns, action_key=none) -%} {# ── Desktop: real table, hidden on phones ── #}
