Why control value is shown as a range

When Idraa reports how much a control is worth, it shows a range (for example, $80k–$190k) rather than a single number. The range is honest about something most risk tools hide: the model has to assume how strongly each control acts on risk, and those assumptions aren’t precisely known. Instead of pretending to a single exact figure, Idraa varies those assumptions across a plausible spread and reports the range of answers you get.

Treat these figures as modeled estimates for comparison and decision-making — not as measured or guaranteed loss savings. They are most useful for ranking controls against each other and sizing investments, not as a promise of exact dollars returned.

“Too close to call”

Sometimes two controls have value ranges that overlap so much the model can’t reliably say which one matters more. Idraa marks these pairs too close to call. It is not saying the controls are worthless — only that, given the uncertainty in the assumptions, their ranking relative to each other isn’t trustworthy. Don’t base a “this control beats that one” decision on a too-close-to-call pair.

Why the typical case sits below the average

You’ll see two related figures: an average (headline) and a typical-case range. The typical-case figure is usually lower than the average, and that’s expected, not a bug.

Cyber losses are skewed: most years are modest, but a few rare, very large losses pull the average upward. The typical case (the middle outcome — the median, or the most likely value for some factors) isn’t dragged up by those rare disasters, so it sits below the average. A wider gap between the two means a more skewed, more disaster-driven loss profile.

For the underlying distributions and the exact mean-versus-median relationship (for a lognormal loss, the average is the median times exp(σ²/2)), see the FAIR methodology primer.

Average and typical, shown side by side

On runs computed after the mean+typical side-by-side release, the value range, the “fair share” attribution, and the “if removed” figure are all computed on the same average (mean) basis as the headline residual-loss and control-value figures — so they are directly comparable to those headline numbers, and to each other. A smaller, muted typical figure is shown alongside each one: it is the central-case counterpart from the section above (the “usual year” view), included to show how skewed the loss profile is — not a second measurement of the same quantity.

  • Average (primary, unmuted) — on the same basis as the headline residual ALE and control-value figures above; this is the number to use for decisions and for comparing against the headline.
  • Typical (muted, paired alongside the average) — the central-case counterpart; a wider gap from the average means a more skewed, more disaster-driven loss profile.
  • Fair share — each control’s allocated portion of the combined modeled risk reduction (a Shapley value); fair shares sum to the total reduction — this is an allocation, not a drop-cost.
  • If removed — the drop-cost counterfactual: how much modeled annual loss would increase if that one control alone were removed from the current portfolio. It deliberately diverges from fair share for redundant or gating controls — see the run page’s “if removed” note for why.

Older runs (computed before this release) show only the typical-case range as the primary figure, exactly as described in the section above — there is no paired average figure on those runs.

“Ranking-stability check skipped”

Testing whether the control ranking holds up under the assumption-spread takes extra simulation work. On very large runs Idraa may not have enough simulation budget to do it, and will say the ranking-stability check was skipped (with the number of draws it managed). The value ranges are still shown; only the “is the ranking stable?” verdict is withheld so it doesn’t report a conclusion drawn from too little data.

Where this shows up

The same ranges, “too close to call” flags, and average-versus-typical framing appear on the run results, the dashboard control-value tile, the executive PDF report, and the downloadable Excel verification workbook — so the story is consistent wherever you read it.
