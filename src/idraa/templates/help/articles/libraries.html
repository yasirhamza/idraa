Overview

Idraa ships with two curated catalogs — the Scenario Library and the Control Library — plus a Framework→FAIR-CAM Crosswalk that grounds control entries in recognised frameworks (CIS Controls, NIST CSF). Together these let you start from ready-made, peer-reviewed content rather than a blank form.

Library values are starting points to review for your context, not definitive figures. Before relying on any catalog entry, check that the threat actor type, asset class, and loss assumptions match your organisation’s situation. See also the Getting started caveats for more on calibration and decision-support limits.

Scenario Library

The Scenario Library (/library) is a browseable catalog of pre-built risk scenarios. Each entry carries a threat actor type, threat event type, asset class, industry tags, and calibrated FAIR input ranges (Threat Event Frequency, Vulnerability, Primary Loss, Secondary Loss).

Use the filter sidebar to narrow by threat actor, threat category, asset class, or industry. The search box matches on entry name. Results are paginated; use the page controls at the bottom to move through the catalog.

Open an entry’s detail page to see its full description and calibrated parameters. From there you can start a new scenario pre-populated with the entry’s values by clicking Use this scenario. The wizard opens with the library values already filled in; you then adjust any input that does not match your specific context. This is the fastest path to a defensible first estimate.

For the mechanics of adjusting inputs once the wizard is open, see Build a scenario. To bulk-import or export library entries as JSON bundles, see Import & export.

Adopt from any industry and tweak

The Scenario Library shows entries from all industries by default. Industry and sub-sector are optional filter sidebar controls — use them to narrow to your sector, or browse across sectors to find scenarios whose threat actor, threat event, and asset class match your situation regardless of the entry’s tagged industry. A professional-services payroll-BEC scenario, for example, is relevant to many sectors even if it was originally curated for a different vertical.

Whichever industry you adopt an entry from, the wizard pre-fills Threat Event Frequency, Vulnerability, Primary Loss, and Secondary Loss directly from that entry’s curated figures — the pre-fill is entry-absolute, not rescaled to your organisation’s revenue tier or size. The curated reference-class loss figure on the entry is the calibration. The entry’s industry tag is advisory only: useful for browsing and filtering, but it has no effect on any pre-filled value. The Impact-page calibration banner in the wizard confirms this: it shows whether Primary Loss / Secondary Loss reflect the entry’s curated reference-class figures or, where you have saved one, your organisation’s override for that entry. All values remain fully editable — library figures are starting points to review and tweak for your specific context. See the Getting started caveats and Build a scenario for guidance on calibrating your inputs.

Control Library

The Control Library (/controls/library) is a companion catalog of ready-made security controls, each tagged with a FAIR-CAM sub-function, control type, reference annual cost, and framework tags (NIST CSF subcategories, CIS Safeguards, ISO 27001 controls).

Browse and filter by FAIR-CAM sub-function, control type, or framework tag. Cards show whether your organisation has already adopted the control so you do not duplicate effort.

Adopt a control (“Add to My Controls”)

Analyst and Admin users can adopt a library control into their organisation’s editable control set by clicking Add to My Controls on any entry card or detail page. Adopting clones a snapshot of the library entry at that version and creates a new control in your organisation’s control list (/controls) with source = Library-derived.

After adopting you are taken to the control’s detail page where you can review and confirm its FAIR-CAM function assignments, adjust cost figures, and tune effectiveness parameters for your environment before linking it to scenarios. Assignments copied from the library start as unconfirmed; confirm each one once you have reviewed it. Re-adopting the same entry is allowed and results in a second independent copy; the “Already adopted” badge on the card flags this to avoid accidental duplicates.

You can also adopt controls directly from the scenario wizard (Step 5 — Controls). When you adopt from inside the wizard, you are returned to the wizard after the adopt completes so you can immediately link the new control to your scenario.

Framework→FAIR-CAM Crosswalk

The crosswalk maps framework controls — CIS Safeguards and NIST CSF subcategories — to FAIR-CAM sub-functions (Threat Event Frequency reduction, Vulnerability reduction, Primary Loss reduction, and so on). This mapping is the evidence layer behind each library control entry’s FAIR-CAM claim: a control that carries a CIS or NIST tag has its claimed FAIR-CAM sub-function validated against the crosswalk at import time. ISO 27001 controls appear as informational tags on control cards (see above), but are not yet part of the validated crosswalk — an ISO 27001 crosswalk is planned.

The crosswalk is canonical and read-only — it reflects the published framework versions seeded into Idraa. It is used internally to validate control library entries and to surface framework tags on control cards. If you are importing custom control library data, ensure the framework tags in your bundle match the seeded framework identifiers and codes.

Scenario→Control Recommendations

When you open a scenario library entry’s detail page, Idraa shows a Recommended controls panel listing the control library entries that the scenario’s curators suggest as most relevant. The same recommendations appear in the scenario wizard’s Controls step (Step 5) for scenarios that were started from a library entry.

Each recommendation shows whether your organisation has already adopted that control. Un-adopted recommendations display an Add to My Controls button so you can adopt them inline without leaving the page. Recommendations are curated by the library authors and scoped to a maximum of six controls per scenario entry — they are suggestions, not a mandatory control set.

Recommendations also appear on the scenario detail page for scenarios created from a library entry, showing only the controls that have not yet been adopted, as a persistent nudge to complete the suggested control set.
