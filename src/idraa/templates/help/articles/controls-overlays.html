What is a control?

A control is any security measure, process, or technology that your organisation has in place (or is considering). In Idraa each control has a yearly cost and one or more FAIR-CAM sub-functions that describe which part of the attack lifecycle the control addresses.

FAIR-CAM groups sub-functions into three families (LEC / VMC / DSC). See FAIR-CAM sub-functions for the full taxonomy.

How controls reduce modelled risk

When you attach a control to a scenario and run an analysis, Idraa passes your control assignments to the FAIR-CAM engine. The engine applies multipliers to Threat Event Frequency (TEF) and/or Vulnerability inside the Monte Carlo simulation — each iteration of the simulation draws from distributions shaped by those multipliers.

This means controls reduce modelled risk in the simulation output, producing a lower residual loss distribution compared with the uncontrolled (inherent) baseline.

Important: controls do not change the inherent Vulnerability input

The Vulnerability you author in the scenario wizard is your estimate of the inherent probability that an attack attempt succeeds before counting your controls. Attaching controls does not overwrite that number — the engine applies a separate multiplier during the simulation so that you always retain the inherent baseline for comparison. See FAIR methodology primer for the full explanation of inherent vs. residual risk.

Adding controls to a scenario

From any scenario detail page, open the Controls tab. You can attach controls you have already created in your controls library, or adopt controls from the curated Control Library. Once attached, the control appears in the scenario’s assigned-controls list and is included in every subsequent analysis run.

To create or manage controls, go to /controls. Analysts and Admins can create, edit, and deactivate controls; Reviewers have read-only access.

What is an overlay?

An overlay is a cross-cutting risk multiplier applied organisation-wide rather than to a single scenario. Overlays carry two multipliers — frequency multiplier and magnitude multiplier — that scale the TEF and loss outcomes of every scenario included in an analysis run.

Typical use-cases include:

  • A macro threat-environment adjustment (e.g., elevated nation-state activity this quarter).
  • An organisation-wide cyber-insurance effect on secondary loss magnitude.
  • A temporary uplift during a high-risk period such as a merger or system migration.

Overlays compound multiplicatively with controls: the simulation applies all active control multipliers and all active overlay multipliers together in each Monte Carlo iteration. Both are recorded in the run snapshot so the report always reflects exactly which overlays were active at the time.

To create or manage overlays, go to /overlays. Only Admins can create, edit, or deactivate overlays; other roles have read-only access.
