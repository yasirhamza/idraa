chart (existing macro — first-party server-rendered SVG)

macros/chart.html emits chart geometry as inline SVG, sourced from services/chart_svg.py's pure geometry functions (epic #547). Theming is pure CSS custom properties (--chart-inherent / --chart-residual / --chart-appetite / --chart-reduction) recomputed by the browser on the data-theme attribute flip — no client-side JS restyle. Crosshair/tooltip/slider/log-toggle hydration + CSV export are wired by static/js/charts.js.

Named macros exported by this file (call-site sample):

  • loss_exceedance_curve(points, tolerance, currency)
  • exceedance_probability_curve(points, tolerance, currency)
  • dual_lec_curve(payload, tolerance, currency)
  • dual_epc_curve(payload, tolerance, currency)
  • headline_ale_with_ci_band(headline, currency)
  • control_effectiveness_bar(rows)
  • risk_comparison_bar(comparison, currency)

No live demo here — charts need real view-model data shapes; consult the calling templates (runs/_results_panel.html, dashboard/index.html).

unit_aware_inputs (existing macro — 119 lines)

macros/unit_aware_inputs.html dispatches on a sub-function's unit type (elapsed_time, currency, probability) to render the matched input widget. Wraps form_field's label/help/error chrome where applicable.

Named macros exported by this file:

  • unit_input(name, unit, value, ...)
  • unit_label_suffix(unit)
  • unit_display(unit, value)

No live demo here — sub-function unit dispatch needs the controls/_assignment_row context.
