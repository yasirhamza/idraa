Field guidance

The CSV template carries no comment lines, so the field rules are here:

  • name — required, unique among active scenarios.
  • scenario_type, threat_category, threat_actor_type, asset_class, status — must be valid enum values (e.g. custom, ransomware, cybercriminals, systems, active).
  • distribution — the distribution shared by the FAIR factors (e.g. PERT).
  • tef_*, vuln_*, pl_*, sl_* — low / mode / high triplets for Threat Event Frequency, Vulnerability, Primary Loss, and Secondary Loss. Each provided triplet must satisfy low ≤ mode ≤ high. Leave a triplet blank to omit that factor (Secondary Loss is optional).
  • JSON uploads use nested objects (threat_event_frequency, vulnerability, primary_loss, secondary_loss) — see the JSON sample.
